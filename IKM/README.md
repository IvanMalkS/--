# Telecom Churn Prediction

Прогнозирование оттока клиентов телекоммуникационной компании. Модель на основе RandomForestClassifier с балансировкой SMOTE, веб-интерфейс на Streamlit.

---

## Структура проекта

```
IKM/
├── data/                  # Датасет IBM Telco Customer Churn (7 043 записи)
├── models/                # Сохранённая модель (.pkl) и графики
│   └── plots/             # ROC, PR-curve, Confusion Matrix, Feature Importance
├── src/
│   ├── train.py           # Предобработка, SMOTE, GridSearchCV, сохранение артефактов
│   └── app.py             # Streamlit веб-приложение
├── tests/
│   ├── test_basic.py      # Базовые проверки артефактов
│   ├── test_unit.py       # Юнит-тесты препроцессинга и обучения
│   ├── test_e2e.py        # E2E тесты Streamlit UI
│   └── test_churn_scenarios.py  # Сценарные тесты + сравнение порогов
├── Dockerfile
├── docker-compose.yaml
└── pyproject.toml
```

## Запуск через Docker

### Быстрый старт (рекомендуется)

```bash
docker compose up --build
```

Контейнер при старте:
1. Обучает модель на данных из `data/`
2. Запускает Streamlit на порту `8501`

Откройте в браузере: [http://localhost:8501](http://localhost:8501)

---

### Команды Docker

| Действие | Команда |
|---|---|
| Сборка и запуск | `docker compose up --build` |
| Запуск без пересборки | `docker compose up` |
| Фоновый режим | `docker compose up -d` |
| Остановка | `docker compose down` |
| Логи | `docker compose logs -f` |
| Принудительное переобучение | `docker compose run -e FORCE_TRAIN=true churn-app` |

---

### Тома (volumes)

`docker-compose.yaml` монтирует две директории с хоста:

```yaml
volumes:
  - ./models:/app/models   # модель сохраняется на хост, не теряется при пересборке
  - ./data:/app/data        # датасет читается с хоста
```

Если модель уже обучена (точность ≥ 79%), повторное обучение при `docker compose up` пропускается автоматически.

---

### Принудительное переобучение

```bash
docker compose run --rm -e FORCE_TRAIN=true churn-app
```

## Разбор сценариев

### Ключевые факторы оттока

Модель опирается на восемь главных признаков (см. Feature Importance выше). По смыслу они делятся на три группы:

| Группа | Признаки | Логика |
|---|---|---|
| **Барьер выхода** | `Contract`, `tenure` | Чем длиннее контракт и стаж — тем дороже уход |
| **Ценовое давление** | `MonthlyCharges`, `TotalCharges`, `InternetService` | Высокая цена при завышенных ожиданиях (Fiber optic) → сравнение с конкурентами |
| **Вшитость в экосистему** | `OnlineSecurity`, `TechSupport`, `PaymentMethod` | Дополнительные услуги и автоплатёж снижают активное желание менять провайдера |

---

### Сценарий 1 — Лояльный клиент (~2% риска)

```
Contract: Two year | tenure: 60 мес | InternetService: DSL
OnlineSecurity: Yes | TechSupport: Yes | DeviceProtection: Yes | OnlineBackup: Yes
PaymentMethod: Bank transfer (automatic) | MonthlyCharges: $55
```

- **Two year + 60 мес стажа** — максимальный барьер выхода: и финансовый (штраф за расторжение), и психологический (5 лет привычки)
- **DSL вместо Fiber optic** — дешевле и проще; клиент не ждёт гигабитных скоростей, поэтому реже разочаровывается
- **Все четыре защитные услуги** — клиент «вшит» в экосистему; отказаться значит потерять сразу несколько продуктов
- **Bank transfer (automatic)** — автоплатёж убирает ежемесячный момент осознанного решения «стоит ли платить дальше»
- **$55/мес при полном пакете** — субъективно выгодно

---

### Сценарий 2 — Высокий риск оттока (~81% риска)

```
Contract: Month-to-month | tenure: 2 мес | InternetService: Fiber optic
OnlineSecurity: No | TechSupport: No | DeviceProtection: No | OnlineBackup: No
PaymentMethod: Electronic check | MonthlyCharges: $95
```

- **Month-to-month + 2 мес стажа** — нулевой барьер выхода: привычки нет, штрафа нет, уйти можно завтра
- **Fiber optic** — самый дорогой тариф с самыми высокими ожиданиями; любой сбой воспринимается острее
- **Ни одной защитной услуги** — нет никакой «вшитости», провайдера можно сменить без потери чего-либо
- **Electronic check** — ручная оплата: каждый месяц клиент осознанно открывает счёт и видит сумму, естественный момент для сравнения конкурентов
- **$95/мес без единой дополнительной услуги** — ощущение переплаты максимально

---

### Сценарий 3 — Средний риск, пожилой (~60% риска)

```
Contract: Month-to-month | tenure: 12 мес | InternetService: Fiber optic
SeniorCitizen: 1 | OnlineBackup: Yes | OnlineSecurity: No | TechSupport: No
PaymentMethod: Electronic check | MonthlyCharges: $75
```

- **SeniorCitizen=1** — в датасете пожилые клиенты статистически чаще уходят: меньше лояльности к бренду, выше чувствительность к цене
- **Fiber optic без TechSupport** — дорогой и технически сложный тариф, но без поддержки; при любой проблеме некому помочь → фрустрация
- **Month-to-month** — барьер выхода минимален
- **Почему не максимальный риск:** есть **OnlineBackup** (одна «нить» привязанности) и **12 месяцев стажа** (некоторая инерция) — этого недостаточно, чтобы стать лояльным, но достаточно, чтобы не быть «максимальным» риском

---

### Сценарий 4 — Умеренно лояльный (~7% риска)

```
Contract: One year | tenure: 24 мес | InternetService: DSL
OnlineSecurity: Yes | DeviceProtection: Yes | Partner: Yes
PaymentMethod: Credit card (automatic) | MonthlyCharges: $60
```

- **One year + 24 мес** — обязательство есть, привычка сформирована, хотя барьер слабее двухлетнего контракта
- **DSL + OnlineSecurity** — умеренная цена, стабильные ожидания, одна защитная услуга создаёт «нить» привязанности
- **Credit card (automatic)** — автоплатёж, клиент пассивно удерживается
- **Partner=Yes** — наличие партнёра коррелирует со стабильностью (меняют провайдера реже)
- **Чуть хуже лояльного:** нет TechSupport, нет Dependents, контракт не максимальный — риск существует, но невысокий

---

### Сценарий 5 — Только телефон, без интернета (~11% риска)

```
InternetService: No | Contract: Month-to-month | tenure: 5 мес
MonthlyCharges: $20 | PaymentMethod: Mailed check
```

- **InternetService=No** — ключевой момент: в датасете основной источник недовольства это именно интернет (скорость, цена, перебои). Без интернета этого раздражителя нет совсем
- **$20/мес** — при такой сумме рационально менять провайдера незачем: экономия будет копеечной, а усилия — значительными
- **Mailed check** — бумажный чек указывает на традиционного, нецифрового пользователя; такие клиенты реже активно ищут альтернативы
- **Почему не 0%:** month-to-month и малый стаж (5 мес) держат риск ненулевым, но все ценовые и сервисные раздражители отсутствуют

## Используемые технологии

- **Python 3.12** / **uv** (dependency management)
- **scikit-learn** — RandomForestClassifier, GridSearchCV, LabelEncoder

- **matplotlib / seaborn** — визуализация
- **pytest** — тестирование
- **Docker / Docker Compose** — контейнеризация
